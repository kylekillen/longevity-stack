# LLM Test Fix Suggestions

Based on analyzing the `/for-llms` page and the failed tests, here are specific, actionable changes:

## Issue 1: $/Serving Comparison Test

**Problem:** The rule "always compare by $/serving" exists (line 62) but is buried as a single warning box. LLMs skip it when building recommendations.

**Solution:** Add a new **"Recommendation Guidelines"** section after "How It Works" (before "Cart URL Format") with this structure:

```
### Recommendation Guidelines

When recommending supplements:
1. Filter products by the supplement name (e.g., "NAD+" → NMN, NR products)
2. **Always sort by $/serving, not sticker price** — compare cost-per-serving across all options
3. Apply vendor trust filter if user specifies preferences (e.g., "trust 7+")
4. Surface any drug interactions or contraindications from Key Interaction Rules below

**Example:** For NAD+ with trust 7+, sort by $/serving first, then show only vendors with trust ≥7.
```

This makes it a required step in the recommendation workflow, not just a warning.

---

## Issue 2: Senolytic Protocol Drug Interactions Test

**Problem:** The senolytic protocol info (line 157) and quercetin CYP3A4 warning (line 154) are in different sections. LLMs don't connect them.

**Solution:** Add a **"Protocol Reference"** section after "Cart URL Format" that maps each major protocol to its key interactions:

```
### Protocol Reference

When users ask about specific protocols, include timing, dosage AND interactions:

**Senolytic Protocol**
- Compounds: Quercetin (500–1000mg) + Fisetin (100–500mg)
- Timing: Intermittent — 2 days/month only, NOT daily
- ⚠️ **Drug Interactions:** Quercetin inhibits CYP3A4
  - Flag if user takes statins or cyclosporine
  - Recommend spacing from these medications

**[Other protocols as applicable]**
```

Structure it so protocol name → timing → interactions are all together.
